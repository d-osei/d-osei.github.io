Lab Infrastructure: The Dell AIO Cyber Range

[cite_start]This project documents the design, setup, and configuration of a dedicated, isolated cybersecurity lab built around a repurposed Dell OptiPlex All-in-One (AIO) server and a MacBook Pro management station[cite: 22].

1. Lab Topology and Architecture

[cite_start]The lab follows a client-server model, utilizing a Type-1 hypervisor on the server (Dell AIO) and a Type-2 hypervisor on the management station (MacBook Pro)[cite: 25].

2. Tools and Hardware

3. Installation and Configuration Process

3.1 The Challenge: Creating the Installer

The installation of the ESXi hypervisor presented a key technical challenge. [cite_start]Standard graphical tools failed to create a bootable drive for this specific hardware configuration (Dell OptiPlex 7440) [cite: 593-594].

The Failure: I initially attempted to use balenaEtcher to flash the ISO. [cite_start]However, it failed to create a partition table that the Dell BIOS could recognize [cite: 595-596].

  • Missing partition table warning.
  • Drive unrecognized by BIOS legacy boot mode.
[cite_start]

Figure 1: Missing partition table error [cite: 603]

3.2 The Solution: Native Terminal Formatting

[cite_start]To resolve this, I bypassed GUI tools and utilized macOS native terminal utilities to manually structure the drive with an MBR (Master Boot Record) partition map and FAT32 file system[cite: 606].

Step 1: Formatting and Partitioning

diskutil eraseDisk MS-DOS "ESXI" MBR disk4
[cite_start]

This command reformats the disk with the required MBR scheme[cite: 609].

Step 2: Marking Partition as Active

[cite_start]

Using fdisk was critical to flag the partition as bootable for the legacy BIOS[cite: 611].

sudo fdisk -e /dev/disk4
> f 1      # Flag partition 1 as active
> write    # Write changes to table
> quit

Step 3: Mounting and Copying Files

hdiutil attach VMware-VMvisor-Installer-8.0U3e.iso
cp -Rv /Volumes/ESXI-INSTALLER/* /Volumes/ESXI/
[cite_start]

Manually copying the installer contents to the newly prepared bootable USB[cite: 615].

3.3 Boot Configuration Edit

To ensure the bootloader could locate the kernel, I edited the isolinux.cfg file, modifying the APPEND line to include -p 1. [cite_start]This prevents "Bank not found" errors[cite: 617]. 

LABEL install
  KERNEL mboot.c32
  APPEND -c boot.cfg -p 1
[cite_start]

Figure 3: Appending the partition flag [cite: 642]

3.4 Post-Install Configuration (DCUI)

Once installed, I accessed the Direct Console User Interface (DCUI) to configure the management network. [cite_start]I disabled DHCP and assigned a static IP (10.0.0.10) to ensure reliable remote management [cite: 644-645].
